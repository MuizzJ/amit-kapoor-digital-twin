How It Works

One question, five hops.

When a visitor types a question, this is the sequence. Total time from Enter to first spoken word: around two seconds.

01 · Ask
Chat Widget
Visitor types a question on the Next.js site and hits send.
→
02 · Retrieve
Pinecone
Vector search returns the top six most relevant chunks from Amit's work.
→
03 · Think
Claude Sonnet 4.6
LLM reads the retrieved context plus system prompt, streams a first-person answer.
04 · Stream
Widget Renders
Tokens appear in the bubble as Claude generates them. Source pills attach at the end.
→
05 · Listen
ElevenLabs
Listen button calls cloned-voice TTS. Audio streams and plays in Amit's voice.
→
06 · Done
Visitor Hears Amit
End-to-end in under two seconds. Zero hallucinations, cited and grounded.

Where Everything Lives

Four services. No custom infra.

Every piece of the stack runs on a managed provider. Nothing to patch, nothing to babysit. Keys live in environment variables that never touch GitHub.

🧠
Pinecone
Vector Database
Holds the 435 embedded chunks of Amit's work. Searches them by semantic similarity whenever a question is asked. Runs on AWS us-east-1, serverless.
  • Index: ask-amit
  • Model: llama-text-embed-v2 (hosted)
  • Namespace: books
✦
Anthropic
Language Model
Generates the actual answer after reading the retrieved chunks plus your voice rules from prompts/ask-amit.md. Streams tokens back to the widget.
  • Model: claude-sonnet-4-6
  • Prompt caching on context block
  • Max output: 600 tokens (short answers)
🎙
ElevenLabs
Voice Cloning
Holds Amit's cloned voice and streams MP3 audio whenever someone clicks Listen on an answer. First byte typically under one second.
  • Voice ID: TvgKRDHpNi1aqml8qok6
  • Model: eleven_turbo_v2_5
  • Output: mp3_44100_128
▲
Vercel
Hosting (planned)
Will host the Next.js site once Hour 6 deploys. Right now it runs locally on your machine at localhost:3001. Git push triggers redeploy.
  • Runtime: Next.js 14, App Router
  • Edge-cached globally
  • Env vars scoped per environment

The Stack

Every layer, named.

If a non-technical stakeholder asks "what is this built on," hand them this table. Nothing here is home-grown or obscure.

Frontend
Next.js 14 · React 18 · Tailwind
Static marketing page plus a floating chat widget with streaming bubbles, source pills, and Listen button.
Backend
Next.js API Routes (Node)
Two endpoints: /api/rag streams Claude, /api/tts streams ElevenLabs audio.
Ingest
Node + pdf-parse + adm-zip
Reads PDF, PPTX, TXT, chunks at 700 words with 100 overlap, upserts to Pinecone with metadata.
Vectors
Pinecone · llama-text-embed-v2
Serverless. Integrated inference means embedding is free server-side, no separate embedding API.
LLM
Anthropic Claude Sonnet 4.6
Streaming with prompt caching on the retrieved context block. Rules live in an editable markdown prompt.
Voice
ElevenLabs · eleven_turbo_v2_5
Professional voice clone of Amit. Streams MP3. First-byte latency around 500ms to 1.5s.
YouTube
youtube-transcript + oEmbed
Pulls auto-captions and title. Writes a formatted .txt into /data ready for ingest.
Hosting
Vercel (planned) · local now
Single vercel deploy ships everything. Env vars managed in the Vercel dashboard.
